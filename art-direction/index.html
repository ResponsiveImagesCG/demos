<picture>-element art-direction demo

This is a test page for the picture element prototype. The demo demonstrates how the <picture>-element can be used for the orientation based art direction use case (particularly the example of the Nokia Meego browser) explained in section 4.1 of the Use Cases and Requirements for Standardizing Responsive Images document.

Author: Shane Hudson / @shanehudson

Instructions:

  1. Load the page large browser window
  2. On large window you will see the landscape browser.
  3. Now narrow your UA window
  4. Reload the page
  5. You now will see another image showing the narrow browser.

Resolutions:

  1. 320px * 249px
  2. 600px * 168px
  3. 900px * 252px

The chrome web browser showing the RICG website.

The source code:


<picture>
    <source media="(max-width: 30em)" src="img/chrome.jpg">
    <source media="(min-width: 30em)  and (max-width: 48em)" src="img/chrome@tablet.jpg">
    <source media="(min-width: 48em)" src="img/chrome@desktop.jpg">

    <!-- fallback img-element if picture-element is not supported -->
    <img src="img/chrome.jpg">
    
    <!-- a11y: providing alternate text -->
    <p>The chrome web browser showing the RICG website.</p>
</picture>
